## 1. 정체성 3축

### 결정

Quotient의 가치 제안은 다음 3축으로 구성한다:

1. **실 자산 분석** (메인) — 사용자가 직접 입력한 보유 자산을 통합·분석
2. **Paper Trading** (Phase 2) — 가상 자금으로 매매 시뮬레이션·백테스트
3. **AI 분석가 = 학습 도우미** — Claude가 실데이터 분석 + 개념 질문 친절히 답변

### Why 3축

| 축 | 핵심 가치 | 페르소나 |
|---|---|---|
| 실 자산 분석 | "여러 증권사 자산을 한 화면에" + "AI에게 질문" | 개발자·파워유저, 본인 자산을 진지하게 분석 |
| Paper Trading | "실 자금 쓰기 전 검증" + "백테스트" | 초보자 입문 + 파워유저 전략 검증 |
| AI 학습 | "개념을 채팅으로 배움" | 모든 사용자, retention ↑ |

### Why 풀 피벗(Paper 단독)이 아닌가

사용자 제안: "Paper Trading 중심 + 교육 콘텐츠"로 전환.
거부 근거 4가지:

1. **타겟 페르소나 충돌** — 블룸버그 톤(정보 밀도·monospace)은 파워유저 도구. 초보자 학습 도구로 가면 톤·UI 전체 재작업.
2. **경쟁 환경** — 한국 시장에서 "통합 분석 + AI"는 거의 무. Paper Trading + 교육은 키움/미래에셋 모의투자, 유튜브 채널 등 레드오션.
3. **수익 모델 약화** — 본인 자산 분석에 ₩14,900 지불 동기 명확. 모의투자에 구독료는 약함.
4. **AI 매매 일기 가치 절반** — 가상 매매 회고는 진짜 매매 회고만큼 가치 없음.

### Why 3축이 충돌하지 않는가

- **실 ↔ Paper** — 모드 분리(`/app/portfolio` vs `/app/paper`). 같은 AI/마켓 데이터 공유. Paper에서 검증한 전략 → 실로 옮기는 funnel 자연스러움.
- **AI ↔ 학습** — 같은 채팅 인터페이스. 시스템 프롬프트에 "개념 질문에는 친절히 교육 답변" 한 줄 추가만으로 통합. 별도 컨텐츠 작성 비용 0.

## 2. 수익률 랭킹 거부 + 알파 카드 변형

### 결정

**전체 사용자 수익률 랭킹·리더보드는 영구 불가**. 후킹은 **알파 카드**(외부 지수 대비 본인 수익률)로 대체.

### Why 풀 랭킹 영구 불가

1. **자기 신고 데이터 신뢰 붕괴**
   - 현재 모델은 "자기 속이면 자기 손해" 논리로 검증 면제 정당화
   - 랭킹 도입 시 — 가짜 매수일·평단으로 1위 등극 가능. 검증 수단 부재.
   - 결과: 랭킹 의미 무효 + 다른 사용자 분석 정확성에 노이즈

2. **규제 회색 지대** (Critical)
   - "상위 5% 포트폴리오 종목 공개" = 불특정 다수 종목 노출 = **투자권유** 영역
   - 한국 자본시장법 — 투자권유대행인 또는 투자자문업 라이선스 필요
   - 개인 사업자 자격 영구 위반

3. **위험 매매 유도** — 학술 연구상 동료 비교 노출 시 위험 매매·과잉 거래 증가. "분석 도구" 정체성과 충돌.

4. **포트폴리오 사이즈 왜곡** — 작은 자금 + 변동성 큰 종목이 % 수익률 유리 → 위험 추구 동기.

### 알파 카드가 안전한 이유

| 항목 | 풀 랭킹 | 알파 카드 |
|---|---|---|
| 비교 대상 | 다른 사용자 | 외부 공개 지수 (KOSPI·SPX·60-40) |
| 종목 노출 | 1위 포트폴리오 공개 | 본인만 봄 |
| 규제 위험 | 투자권유 영역 | 정보 제공 (개인 분석) |
| 후킹 효과 | 강함 | 강함 (외부 벤치마크 대비도 동기 부여) |

### 변형 카드 추가 옵션 (모두 안전)

- 개인 시계열 (과거의 본인 vs 현재의 본인)
- 익명 분포 ("당신은 상위 N%" — 1위·종목 노출 없음)
- 목표 달성률 ("1년 1억 모으기 → 65%")

## 3. AI 매매 일기 — 핵심 차별화로 격상

### 결정

**Phase 2 최우선 차별화 카드**로 격상. 실/Paper 양쪽 모두 적용.

### Why 핵심 차별화인가

- 한국 핀테크에 없는 기능
- Claude의 추론·패턴 분석 강점 활용 (다른 LLM도 가능하나 우리는 이미 Sonnet 4.6 통합 완료)
- 규제 무관 — 개인 일기 분석은 자문업·권유업 무관
- 본 서비스 정체성("AI 분석가") 강화
- 실/Paper 양쪽 데이터 통합 → AI가 "Paper에서 8월 패턴이 실에서도 반복됨" 같은 메타 분석 가능

### How

- 매매 입력 시 "이유 메모" 필드 (선택)
- AI 채팅에 도구 `analyze_journal` 추가 — 최근 N건의 매매 + 시점 시세 변동 → 패턴 분석
- 예시 출력: "8월에 3번 모두 KOSPI 하락 직후 매도 → 손절 후 반등 놓침 (감정적 매도 패턴)"

### 우선순위 근거

| Phase 2 항목 | 차별화 강도 | 구현 비용 | 우선순위 |
|---|---|---|---|
| **AI 매매 일기** | ⭐⭐⭐ (한국 최초) | 중 (1 테이블 + 1 도구 + UI) | 1 |
| Paper Portfolio | ⭐⭐ (있는 사례 있음) | 큼 (모드 분리·시뮬레이션 엔진) | 2 |
| 알파 카드 | ⭐⭐ | 작음 (1 카드 + history API) | 3 (먼저 출시) |
| CSV import | ⭐ (편의) | 큼 (LLM 파싱) | 4 |

## 4. 자기 신고 데이터 안내 의무

### 결정

사용자 입력 데이터의 **검증 부재**를 다음 두 곳에 명시 안내:
1. 자산 추가 모달 (`AddHoldingDialog`)
2. 온보딩 wizard step 3 (`HoldingsStep`)

문구(공통):
> "본 서비스는 증권사 자동 연동 없이 본인이 직접 기록하는 **개인 분석 도구**입니다. 실제 보유 여부는 검증하지 않으며, 분석 정확성은 입력 정확성에 따릅니다."

### Why 의무인가

- 사용자가 검증 시스템으로 오해할 위험 — 토스·뱅크샐러드 사용 경험에서 "자동" 기대
- "맘대로 입력하는 사람" 차단보다 **기대 설정** 목적 (자기 검증 의식 환기)
- 향후 분쟁/CS 시 책임 소재 명확화

### 어디에 추가하지 않는가

- 설정 페이지 — 매번 보면 피로
- 헤더·푸터 — 노이즈
- 랜딩 페이지 "신뢰 카드" 섹션에 이미 "마이데이터 X" 명시

## 5. AI 분석가 = 학습 도우미 통합

### 결정

별도 교육 컨텐츠(블로그·뉴스레터·강의)는 **만들지 않는다**. AI 채팅이 학습 인터페이스 역할을 겸한다.

### Why 별도 컨텐츠 거부

- 지속 작성 비용 큼 (1인 운영)
- 한국 시장 무료 컨텐츠 풍부 (키움 아카데미·유튜브)
- 정적 컨텐츠 작성 시 "투자 조언" 회색 지대 발생 가능
- 본 서비스 정체성(분석 도구)과 결 다름

### How

시스템 프롬프트에 다음 한 줄 추가:
> "사용자가 PER·ROE·분산 투자·달러 코스트 평균 같은 일반 금융 개념을 물으면 친절히 교육적으로 답한다. 단, 특정 종목 매수/매도 권유는 절대 금지."

이 한 줄로 별도 컨텐츠 페이지 없이 학습 기능 완성.

## 6. 랜딩 페이지 구조 결정

### 결정

10 섹션 단일 페이지(scroll). 본 서비스 톤(블룸버그·monospace·1px 라인) 유지하면서 정보 밀도 강화.

| # | 섹션 | 역할 |
|---|---|---|
| 1 | Live ticker bar | "살아있는 사이트" 인상 |
| 2 | Hero | 3축 가치 명시 + CTA |
| 3 | Dashboard preview | 시각 데모 (홈 6 카드 SVG mock) |
| 4 | AI Chat preview | USP 시각화 (도구 호출 인디케이터 포함) |
| 5 | Feature grid (4) | INTEGRATE·ANALYZE·MONITOR·BRIEF |
| 6 | Paper Trading teaser | Phase 2 기대감 형성 |
| 7 | Trust cards (4) | 안 하는 것 명시 (신뢰) |
| 8 | FAQ (4) | 검증·차별점·AI·Paper |
| 9 | Bottom CTA | 가입 버튼 반복 |
| 10 | Footer (4 컬럼) | LEGAL·CONTACT·STATUS |

### Why 이 순서인가

- 진입자 흐름: 살아있음(ticker) → 무엇인가(hero) → 어떻게 생겼나(preview) → 차별점(chat) → 상세(feature) → 미래(paper) → 안전한가(trust) → 의심 해소(FAQ) → 행동(CTA)
- 각 섹션 하단에 다음 섹션으로 자연 유도
- Footer까지 안 가도 Bottom CTA에서 가입 완결
